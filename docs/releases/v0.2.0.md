# v0.2.0

## Changelog

- 新增 workspace env 配置，支持把普通变量和从本机环境变量读取的 secret 合并写入 `${workspace}/.env`。
- 新增 workspace 依赖安装步骤，支持在 workspace 或 workspace 子目录中执行多条安装命令。
- 调整 recover 触发条件：network policy 应用后不再触发 recover；recover 仅用于需要重新加载的 channel 配置。
- 部署计划和完成摘要会展示 env key、secret 来源、依赖命令和执行结果。
- 加强日志脱敏，额外处理 npm token 和常见 `TOKEN` / `SECRET` / `PASSWORD` 赋值。

## 尚未实现

- JSON 输出格式尚未定义。
- 结构化日志目标格式尚未定义。
- 依赖安装命令仍由用户显式配置，尚未自动识别 Node/Python/Go 等项目类型。
- env 文件冲突策略目前固定为按 key 合并更新，尚未提供 backup/overwrite 选项。
